# 报价不自推：任何价格断言必须有可验证的市场锚点
type: method
date: 2026-07-08
source: 1688开店服务定价——Agent 自推 ¥1,688 被平台人员驳回，根因是成本加价法推理而非找真实市场数据

## 原则

Agent 本能用成本加价法（拆解→估每个组件成本→加总）。这是错的。市场价格是竞争均衡结果，不是成本累加。

正确做法：**任何价格断言必须有可验证的公开来源作为锚点。** 没有锚点 → 标注"估算"，不包装成事实。

## 步骤

1. **找到这个品类的真实交易市场** — 不是搜淘宝/百度，是搜"这个品类实际在哪里买卖"。B2B服务→平台服务市场（如 fuwu.1688.com），SaaS→官网定价页，自由职业→Upwork/Fiverr 公开 rate card，实物商品→平台在售 listing
2. **取锚定价（市场底价）** — 找公开可见、可被用户验证的最低价格。这个锚点可能比你便宜还比你全——必须承认
3. **从锚点往上加，每一项可解释** — 你的报价 = 锚定价 + 增量价值。每个差价必须有对应的可感知增量。如：锚定价只给模板自动上架，你加了人工筛选→差价 ¥500 = 人工把关

## 为什么不能自己推理

- Agent 用成本加价法推理价格是系统性问题——不只是 1688，任何涉及定价的任务都会触发这个本能
- 平台生态内的服务商经过市场竞争，报价是出清结果，不是成本加价
- 用户或对手也能搜到同一个公开锚点，你的报价合不合理一眼就知
- 自推价格被戳穿 → 信任崩塌。承认"没找到锚点"比虚报价格强

## 预防

- 涉及任何报价/定价任务 → 先搜"这个品类实际在哪里交易"，找到公开价格页
- 报价必须有可验证来源。一个 URL 胜过一千字推理
- 找不到公开锚定价 → 标注"未找到公开锚定价，基于 XX 推理"，不让推理冒充事实
- 汇报时锚点在前，推理在后——用户先看到真实市场数据，再看到你的判断
