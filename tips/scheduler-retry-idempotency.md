# 调度器重试导致 agent 副作用重复

type: architecture
date: 2026-07-08
source: 个体户申报提醒同一天发了两遍飞书消息

## 现象
cron job 第一次运行成功发送飞书通知，但没写产出文件 → 调度器标记 output_missing → 自动重试 → agent 又发了一遍消息。

## 根因
调度器的重试机制和 agent 的副作用之间没有幂等性协调。agent 不知道自己在被重试，每次执行都完整走一遍（包括发消息/通知/部署等副作用）。

## 修复

### 调度器层（框架级）
1. **注入重试上下文**: 重试时在 agent prompt 前自动拼接"这是第 N 次重试，上次失败原因: X，如果副作用已完成则跳过只写产出文件"
2. **重试时不删旧产出文件**: 首次运行删文件确保新鲜，重试时保留让 agent 能检测到"已执行过"

### Job 层（防御性）
3. Prompt 内加幂等检查: "先读今天的产出文件，存在则说明已执行过，直接退出不重复发消息"

### 为什么不是只改 prompt
框架级注入覆盖所有 agent job，新 job 上线自带重试安全。每个 prompt 各自写幂等检查 = 迟早漏一个。

## 三层防护
1. 调度器注入上下文（所有 agent job）
2. 保留旧产出文件（物理信号）
3. prompt 自身幂等检查（防御性）
