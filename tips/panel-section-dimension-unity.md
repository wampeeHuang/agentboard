# 面板板块必须同一维度，混维用户无法识别界面结构

type: method
date: 2026-07-08
source: Inspector panel.html 板块重组——用户两次指出板块标题无区分度、内容识别性差

## 现象

Dashboard 面板有 10 个区域（总览卡片、统计筛选、操作按钮、项目网格、Cron 摘要、工具摘要、设计原则……），用户打开后反馈"每一部分每一个板块都应该有它的标题，不然无法识别每个板块是干什么的"。

加了标题后用户进一步指出"骨/肉标题跟板块标题根本没区分开来"——根因不在缺标题，在板块切分维度是混的。

## 根因

三个维度混在同一层：

| 区域 | 按什么切 |
|------|---------|
| A 三源总览 | 数据来源（Inspector/scheduler/agentboard） |
| 统计卡片 + 按钮 | 操作动作（筛选/触发巡检） |
| B 项目卡片 | 被观察对象（骨项目/肉项目） |
| C Cron 摘要 | 数据来源 |
| D 工具摘要 | 数据来源 |
| 设计原则 | 元信息 |

三个维度的标题放在同一视觉层级，读者无法建立心智模型——到底在看什么？

## 修复

1. **选一个维度做顶层**。Inspector 场景选了"被观察对象"——用户打开面板看三样东西的健康：项目、任务、工具。
2. **每板块自成闭环**：状态灯 → 统计 → 明细，一条线走到底。
3. **原 A 区总览灯拆散归位**——每个板块的 section-head 自带状态灯，不单独浮一层。

新结构（4 板块，同一维度）：

```
§ 项目巡检   ● 状态灯 + 筛选卡片 + 操作按钮 + 项目网格
§ 定时任务   ● 状态灯 + Cron 明细
§ 在线工具   ● 状态灯 + 工具明细按 category
§ 设计原则   [元信息，不是数据]
```

## 预防

建面板/仪表盘前先问：**顶层板块按什么切？** 只能有一个答案。数据来源、操作动作、被观察对象——选一个。其余维度降级为板块内标签或筛选器。

判断标准：随便指一个板块，用户能一句话说清"这是什么"——不需要先理解其他板块的维度。
