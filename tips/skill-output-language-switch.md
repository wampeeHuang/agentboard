# 技能产出换语言：翻译已有产出，不重跑管线

type: method
date: 2026-06-29
source: codebase-to-course 中文版——英文 HTML 模块已存在且结构正确，却重跑了整个 brief→agent→assemble 管线

## 现象
- 技能（如 codebase-to-course）的产出物已经生成完毕，结构正确、交互元素齐全
- 用户要求换语言（如"我要中文"）
- 自然地按技能流程重跑：重写 briefs → 重新分派 agent → 等全部完成 → 重新组装
- 6 个 agent × ~100K tokens 每个 + 等待时间，而翻译只需读 HTML 改文字

## 根因
技能流程是为"从零创建"设计的，不是为"修改已有产出"设计的。当用户要求修改已有产出时，技能流程的惯性会把思路带向"重建"而非"编辑"。

技能是工具，不是任务——用户要的是中文 HTML，不是"重新执行技能"。

## 步骤
已有产出物换语言的标准路径：

1. 确认现有产出物结构正确、无需改动
2. 直接读已有文件，用 Edit 工具替换文字内容
3. 代码片段、变量名、文件路径保持原文
4. 重新组装（如有需要）并验证

**适用场景：** 任何技能生成的 HTML/Markdown/文档类产出，换语言、调风格、改比喻，只要结构不动就走编辑，不走重建。

## 预防
- 用户说"换成中文""改成XX风格"时，先问：现有文件结构对不对？对 → 编辑；不对 → 重建
- 技能流程的"Phase 1-4"是为首次创建设计的，修改已有产出时不套用
- token 成本自检：编辑 1 个文件 < 分派 1 个 agent < 分派 N 个 agent
